## 2. Agent 執行規範與核心要求 (Agent Execution Guardrails)

* **視覺不跑調原則**：全站禁止使用任何超出色彩定義系統（如自定義的藍、紅、立體陰影或圓潤的大圓角 `rounded-2xl`）。一律套用 `#05070A` (背景) 與 `#00FF66` (焦點綠)。
* **Vibe Coding 狀態一致性**：每一次修改或擴充新功能（如點擊按鈕的錨點滾動），均需維持代碼結構的乾淨，不可為了特定功能引入無關的外部 JS 套件，盡可能以純 HTML/Tailwind CSS 與原生 JavaScript 實作。
* **動態進度回報機制**：
    * Agent 每完成一個主要階段（如完成前端切版、問卷串接等），**必須主動重寫或更新本文件**。
    * 將對應任務的狀態從 `[ ] 未開始` 或 `[>] 進行中` 改為 `[X] 已完成`。
    * 並在完成時間與回報日誌欄位留下詳細技術記錄。
